# VoxProof Enterprise Technical Whitepaper

## Executive Summary

VoxProof is a runtime security gateway and adversarial testing platform for AI voice agents.

The platform combines:

* Real-time conversational threat detection
* Voice-layer security heuristics
* Prompt injection classification
* Tool-call policy enforcement
* AI-assisted risk analysis using Gemini
* Deterministic policy inspection via LobsterTrap
* Replayable adversarial attack suites
* Compliance-oriented reporting

Unlike generic AI observability systems, VoxProof is purpose-built for voice and conversational agents operating in high-risk enterprise environments.

The repository implements a modular FastAPI backend, React/TypeScript frontend, deterministic security engines, replay orchestration infrastructure, and enterprise-oriented reporting workflows.

## Responsibilities

* Simulate attack transcripts
* Run security pipeline

* Generate trust decisions
* Produce reports

# RAGSanitizer

## Purpose

Sanitize retrieval-augmented generation context before injection into LLM prompts.

## Threats Addressed

* Malicious embedded instructions
* Poisoned documentation
* Embedded prompt overrides
* Retrieval-based jailbreaks

## Enterprise Importance

RAG poisoning is becoming one of the most important enterprise AI attack vectors.

The sanitizer layer significantly improves operational safety.

# BoundaryEngine

## Purpose

## Boundary Categories

| Boundary       | Description               |

## Architectural Value

* explainability,
* compliance mapping,

# GeminiAdapter

## Purpose

Integrate Google Gemini models for semantic security analysis.

## Capabilities

| Capability            | Description               |
| --------------------- | ------------------------- |
| Risk Classification   | Threat category detection |
| Root Cause Analysis   | Failure explanation       |
| Confidence Scoring    | AI certainty estimation   |
| Suggested Remediation | Fix recommendations       |

## Runtime Optimization

The repository parallelizes Gemini calls during replay execution.

This reduces execution time from minutes to seconds.

This is explicitly referenced in the backend implementation.

* Policy violations
* Number of findings
